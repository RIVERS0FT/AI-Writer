## 二十六、主要风险

### Tauri Mobile 成熟度

桌面端应作为首发目标。Android 和 iOS 虽然可以构建，但必须实际验证：

- Tiptap 长文本性能
- 软键盘行为
- 中文输入法
- 应用切到后台后的生成任务
- 插件移动端支持程度
- 文件导入导出
- WebView 差异

### WebView 差异

Windows、macOS、Linux、Android 和 iOS 使用的 WebView 实现不同，富文本编辑器需要做跨平台测试。

### Rust 学习成本

团队至少需要一名成员掌握：

- Rust 基础
- async
- serde
- error handling
- SQLite
- Tauri Command
- Tauri Plugin

但不需要将所有业务逻辑迁移到 Rust。

### 长篇上下文成本

必须从初期就设计摘要、检索、Token 预算和版本管理，不能等小说变长后再补。

## 二十八、方案结论

AI-Writer 最合适的整体架构是：

```text id="u7ovg6"
React 负责界面
TypeScript 负责小说领域逻辑
Rust 负责本地数据、安全和系统能力
SQLite 负责本地持久化
Tauri 负责桌面与移动应用容器
PWA 负责 Web 平台
Python 只作为可选高级扩展
```

开发顺序应当是：

```text id="p4kg7j"
桌面版核心功能
    ↓
稳定小说生成工作流
    ↓
平台能力抽象
    ↓
Android 原型验证
    ↓
iOS 和 Web
    ↓
云同步及高级功能
```

这套方案既保留了 React、TypeScript 和 Tiptap 的成熟文本编辑生态，也通过 Tauri/Rust 获得本地数据库、密钥保护、文件系统和跨平台发布能力，适合作为 AI-Writer 的长期技术基础。
