# 纯文本编辑器架构决策

## 决策

AI-Writer 不支持富文本编辑。该决策覆盖早期设计中的 Tiptap、ProseMirror 和富文本主格式建议。

## 原则

- 章节正文以 `plain_text` 为唯一内容来源。
- 编辑器使用浏览器原生多行文本输入。
- AI 生成、改写、恢复和版本保存只处理纯文本。
- 人物、世界观、大纲、记忆和 Prompt 也使用纯文本。
- Markdown 只用于未来导出，不作为内部编辑格式。

## 数据模型

章节与章节版本各自只保留一个正文列：

```text
chapters.plain_text
chapter_versions.plain_text
```

TypeScript 类型、Zod 输入、SQLite 仓储、Web 本地存储和界面调用统一使用 `plainText`。不再存在 JSON 正文或 Markdown 正文字段。

## 历史数据升级

SQLite 第三版迁移先保留已有纯文本，在纯文本为空时使用旧文本列补充数据，然后删除两个旧列。Web 端读取旧本地记录时也会写回只含 `plainText` 的新对象。

## 影响

不再需要 Tiptap 及其 Starter Kit。选区改写、查找替换、字数统计、自动保存和版本比较均基于普通字符串实现。
