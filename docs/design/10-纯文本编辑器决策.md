# 纯文本编辑器架构决策

## 决策

AI-Writer 不支持富文本编辑。该决策覆盖并替代早期设计文档中关于 Tiptap、ProseMirror 和富文本主格式的建议。

## 原则

- 章节正文以 `plain_text` 为唯一权威内容。
- 编辑器使用浏览器原生多行文本输入能力。
- AI 生成、改写、恢复和版本保存都只处理纯文本。
- 人物、世界观、大纲、记忆和 Prompt 内容也使用纯文本。
- 不保存字体、颜色、字号、加粗、链接、内嵌图片或其他富文本样式。
- Markdown 仅作为未来导出格式，不作为编辑器内部格式。

## 兼容字段

现有数据库包含 `content_json` 和 `content_markdown`。在数据库重构前：

- `content_json` 保存纯文本兼容文档。
- `content_markdown` 同步保存与 `plain_text` 相同的文本。
- 新代码不得将富文本结构写入这些字段。

## 旧数据迁移

打开旧章节时，应用会将常见的旧编辑器标记转换为换行和纯文本。转换后的内容在下一次自动保存时写回数据库，从而完成渐进迁移。

## 影响

不再需要 Tiptap 及其 Starter Kit 依赖。选区改写、查找替换、字数统计、自动保存和版本比较将基于标准文本选择范围与字符串操作实现。
