## 五、富文本编辑器

推荐使用 Tiptap。

Tiptap 基于 ProseMirror，并提供 React 集成，适合构建可扩展的编辑器。([tiptap.dev](https://tiptap.dev/docs/editor/getting-started/install/react))

AI-Writer 的编辑器应支持：

- 标题和正文
- Markdown 快捷输入
- 文本选区
- AI 改写选区
- 扩写和缩写
- 批注
- 查找替换
- 字数统计
- 自动保存
- 历史版本
- AI 生成内容标记
- 修改前后差异比较

正文建议同时保存三种形式：

```text id="60k663"
content_json       Tiptap JSON，作为主格式
content_markdown   用于导出和迁移
plain_text         用于模型上下文和全文检索
```

不要只保存 HTML，因为 HTML 不适合作为长期、稳定的编辑器数据格式。

## 六、TypeScript 领域层

小说核心逻辑应放在独立的 TypeScript 包中，而不是写在 React 组件里。

```text id="hsv1la"
packages/core
├── entities
├── workflows
├── context
├── prompts
├── providers
├── consistency
├── token-budget
└── errors
```

核心领域对象包括：

```ts id="fgaep5"
interface NovelProject {
  id: string;
  title: string;
  genre: string;
  summary: string;
  status: "planning" | "writing" | "completed";
}

interface Chapter {
  id: string;
  projectId: string;
  volumeId?: string;
  title: string;
  order: number;
  status: "planned" | "drafting" | "completed";
  content: EditorContent;
  summary?: string;
}

interface Character {
  id: string;
  projectId: string;
  name: string;
  profile: string;
  motivation: string;
  currentState: string;
}
```

该层不能直接依赖：

- Tauri
- 浏览器 localStorage
- SQLite 插件
- Android API
- Node.js 文件系统

这样才能在桌面端、移动端和 Web 端共享。

## 七、平台抽象层

定义统一平台接口：

```ts id="8p3hxv"
interface PlatformService {
  database: DatabaseService;
  secureStorage: SecureStorageService;
  filesystem: FileSystemService;
  http: HttpService;
  notifications: NotificationService;
  updater?: UpdaterService;
}
```

分别实现：

```text id="uar49a"
TauriPlatformService
WebPlatformService
```

例如安全存储：

```ts id="5ry7s1"
interface SecureStorageService {
  setSecret(key: string, value: string): Promise<void>;
  getSecret(key: string): Promise<string | null>;
  removeSecret(key: string): Promise<void>;
}
```

Tauri 版本使用 Stronghold 或系统安全存储，Web 版本原则上不长期保存 API Key，或者使用经用户密码派生密钥加密后的 IndexedDB。
