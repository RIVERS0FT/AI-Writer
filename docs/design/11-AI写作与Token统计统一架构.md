# AI 写作与 Token 统计统一架构

## 目标

AI-Writer 的模型写作请求统一进入写作任务和编排体系，不再由界面分别管理规划、重试、审查和用量。Token 只用于统计、审计和模型容量判断，不作为应用额度或停止条件。

## 统一调用链

```text
UI 写作操作
  -> WritingOrchestrator
  -> GenerationJob
  -> ContextBuilder
  -> WritingStep
  -> ProviderRuntime
  -> GenerationRequest
  -> TokenUsage
  -> GenerationOutput
```

当前章节续写执行链：

```text
构建项目上下文
  -> 章节规划
  -> 生成新增正文
  -> 连续性、人物与风格联合审查
  -> 必要时定向改写
  -> 返回最终新增正文
  -> 现有保存链写入章节与任务输出
```

后续将在同一编排器中加入分场景生成、保存步骤和记忆候选提取。

## 写作任务

每个任务记录：

- 项目和章节
- 写作任务类型
- 用户写作要求
- Provider 与模型档案
- 流水线版本
- Prompt 集版本
- 各阶段开关
- 汇总 Token
- 重试次数
- 开始、完成和错误状态

流水线阶段开关只表示用户或工作流是否需要该步骤，不根据 Token 用量自动关闭。

## 写作步骤

每个步骤独立保存：

- 步骤类型和顺序
- 状态和尝试次数
- Prompt ID 与版本
- 输入与输出结构
- 输入、输出和总 Token
- 用量来源
- 延迟
- 上下文快照引用
- 开始、完成和错误信息

已经接入：

- `context_build`
- `chapter_plan`
- `draft`
- `continuity_review`
- `targeted_rewrite`

审查没有发现需要修订的问题时，`targeted_rewrite` 会保存为 `skipped`，并记录跳过原因。

## Prompt 流水线

当前版本化 Prompt：

```text
chapter-planner@1
chapter-draft-writer@1
continuity-reviewer@1
targeted-rewriter@1
```

规划和审查要求模型返回 JSON。规划失败时任务失败并保留步骤错误；审查 JSON 无法解析时不会盲目重写，而是保存跳过原因。

正文与修订结果始终为纯文本。

## 上下文构建

编排器从以下来源构建上下文：

1. 项目简介
2. 锁定人物资料
3. 未锁定人物资料
4. 锁定世界观
5. 未锁定世界观
6. 当前章节和其他大纲节点
7. 最近章节摘要或正文尾部
8. 当前章节完整正文

人物资料包含别名、人物设定、核心动机和当前状态。锁定资料始终高于普通资料。

## 请求级用量

每次 Provider 请求都创建独立请求记录，包括规划、初稿、审查、改写及其重试请求。请求记录保存：

- 任务和步骤
- Provider、模型档案与模型 ID
- 请求尝试序号
- 请求状态
- 输入、输出和总 Token
- 缓存输入 Token 与推理 Token
- 精确、估算或未知来源
- 延迟和错误
- 开始与完成时间

任务汇总 Token 是请求明细的派生缓存，不能代替请求明细。步骤汇总只统计属于该步骤的请求，避免后续步骤重复累计前序 Token。

## 未知值语义

- Provider 返回 Token：`provider`
- 由 tokenizer 或启发式算法计算：`estimated`
- 无法获得：`unknown`
- 未知字段保存为 `NULL` 或 `undefined`
- 不得使用零表示未知

## 自动重试

每次自动重试：

1. 创建新的请求 ID。
2. 增加 attempt。
3. 独立记录延迟、状态和 Token。
4. 不覆盖先前失败请求。
5. 成功后分别汇总当前步骤和整个任务用量。
6. 恢复到当前步骤对应的任务状态，而不是统一恢复成正文生成状态。

统计写入失败不能中断正文生成。

## 上下文容量

项目不使用固定比例 Token 预算。上下文构建默认保留全部符合检索和权限规则的资料。

只有预计输入加请求输出超过模型真实上下文窗口时，容量管理器才进行兼容处理：

1. 锁定资料全部保留。
2. 其余资料按相关优先级排序。
3. 从低优先级资料开始省略，直到适配模型窗口。
4. 上下文步骤记录预计 Token、省略条目数和是否触发容量处理。

这属于模型技术兼容，不属于用量限制。

## 存储

SQLite 第四版迁移扩充：

- `generation_jobs`
- `generation_steps`
- `generation_requests`
- `usage_records`

Web 使用对应的 localStorage 仓储。请求明细是统计事实来源，任务和步骤 Token 是汇总缓存。

## 当前限制

- 现有 UI 仍保留旧 Prompt 字符串作为兼容参数，但运行时编排器会替换为版本化 Prompt；后续会删除这些无效字符串。
- 当前按整章新增正文生成，尚未拆成多个 `scene_plan` 和场景草稿步骤。
- 联合审查使用一个请求，人物、风格和连续性尚未拆分为独立审查器。
- 最终章节保存仍由现有 AppShell 保存链完成，尚未迁入 `save` 步骤。
- 尚未实现 `memory_extraction`。
- 尚未实现 Token 统计管理页面。

## 后续阶段

1. 删除 UI 内不再生效的旧 Prompt 拼接。
2. 实现分场景计划和逐场景生成。
3. 将人物、连续性和风格审查拆成独立步骤。
4. 将章节保存和版本快照迁入编排器。
5. 实现章节完成后的记忆候选提取。
6. 增加任务、章节、项目和模型维度统计界面。
