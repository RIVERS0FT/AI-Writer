# AI 写作与 Token 统计统一架构

## 目标

AI-Writer 的所有模型写作请求统一进入写作任务体系，不再由界面分别管理任务、重试和用量。Token 只用于统计、审计和模型容量判断，不作为应用额度或停止条件。

## 统一调用链

```text
UI 写作操作
  -> GenerationJob
  -> WritingStep
  -> ProviderRuntime
  -> GenerationRequest
  -> TokenUsage
  -> GenerationOutput
```

后续完整编排链为：

```text
构建上下文
  -> 章节规划
  -> 场景生成
  -> 一致性审查
  -> 定向改写
  -> 保存正文
  -> 记忆候选提取
```

## 写作任务

每个任务记录：

- 项目和章节
- 写作任务类型
- 用户写作要求
- Provider 与模型档案
- 流水线版本
- Prompt 集版本
- 各阶段开关
- 汇总 Token
- 重试次数
- 开始、完成和错误状态

流水线阶段开关只表示用户或工作流是否需要该步骤，不根据 Token 用量自动关闭。

## 写作步骤

每个步骤独立保存：

- 步骤类型和顺序
- 状态和尝试次数
- Prompt ID 与版本
- 输入与输出结构
- 输入、输出和总 Token
- 用量来源
- 延迟
- 上下文快照引用
- 开始、完成和错误信息

首个实现阶段已经将现有章节续写映射为 `draft` 步骤。后续规划、审查和记忆提取会使用同一结构。

## 请求级用量

每次 Provider 请求都创建独立请求记录，包括重试请求。请求记录保存：

- 任务和步骤
- Provider、模型档案与模型 ID
- 请求尝试序号
- 请求状态
- 输入、输出和总 Token
- 缓存输入 Token 与推理 Token
- 精确、估算或未知来源
- 延迟和错误
- 开始与完成时间

任务汇总 Token 是请求明细的派生缓存，不能代替请求明细。

## 未知值语义

- Provider 返回 Token：`provider`
- 由 tokenizer 或启发式算法计算：`estimated`
- 无法获得：`unknown`
- 未知字段保存为 `NULL` 或 `undefined`
- 不得使用零表示未知

## 自动重试

每次自动重试：

1. 创建新的请求 ID。
2. 增加 attempt。
3. 独立记录延迟、状态和 Token。
4. 不覆盖先前失败请求。
5. 成功后重新汇总整个任务用量。

统计写入失败不能中断正文生成。

## 上下文容量

项目不再使用固定比例 Token 预算。上下文构建默认保留全部符合检索和权限规则的资料。

只有预计输入加请求输出超过模型真实上下文窗口时，容量管理器才触发压缩。压缩属于模型兼容处理，不属于用量限制。

始终优先保留：

- 当前用户要求
- 锁定人物和世界观
- 当前章节及场景计划
- 当前人物状态和知识
- 硬约束

## 存储

SQLite 第四版迁移扩充：

- `generation_jobs`
- `generation_steps`
- `generation_requests`
- `usage_records`

Web 使用对应的 localStorage 仓储。请求明细是统计事实来源，任务和步骤 Token 是汇总缓存。
