# AI-Writer：Tauri 2 + React 全平台技术方案

## 一、方案定位

AI-Writer 是一款用户配置大模型 API 后，自动辅助或自动完成小说创作的应用。

主要能力包括：

- 配置 OpenAI、Claude、Gemini、DeepSeek、Ollama 等模型
- 创建小说项目
- 生成人物、世界观和大纲
- 自动规划章节
- 流式生成章节正文
- 维护人物状态、时间线和伏笔
- 检查剧情一致性
- 保存章节历史版本
- 导出 Markdown、TXT、DOCX 等格式
- 在桌面、手机和平板上使用

推荐的整体技术组合是：

```text id="axgtkk"
应用容器：Tauri 2
前端框架：React + TypeScript
构建工具：Vite
样式系统：Tailwind CSS
UI 组件：shadcn/ui
富文本编辑：Tiptap
前端状态：Zustand
服务端状态：TanStack Query
数据校验：Zod

本地原生层：Rust
本地数据库：SQLite
机密信息：Tauri Stronghold
模型请求：Rust HTTP 或 TypeScript Provider
桌面后台扩展：可选 Python Sidecar
Web 版本：React PWA
```

Tauri 2 可以使用编译为 HTML、JavaScript 和 CSS 的前端框架，并面向主流桌面和移动平台构建应用。它使用操作系统已有的 WebView，而不是像 Electron 一样在每个应用中附带完整浏览器内核。([v2.tauri.app](https://v2.tauri.app/start/))

## 二、目标平台

该方案可以覆盖：

| 平台 | 实现方式 |
|---|---|
| Windows | Tauri 桌面应用 |
| macOS | Tauri 桌面应用 |
| Linux | Tauri 桌面应用 |
| Android | Tauri Mobile |
| iOS | Tauri Mobile |
| Web | React PWA 独立构建 |

其中：

```text id="mtkoym"
Windows、macOS、Linux、Android、iOS
    共用 React 前端
    共用 TypeScript 业务逻辑
    共用大部分 Rust 核心代码

Web
    共用 React 前端和 TypeScript 业务逻辑
    使用浏览器平台适配层
```

Tauri 本身不负责将程序发布为普通网站。Web 版本由同一套 React 项目单独执行 Vite 构建，生成 PWA。

实际代码复用目标建议设为：

```text id="3iy5y2"
业务逻辑：90% 以上
React 组件：80%～90%
数据库模型：80% 以上
平台原生代码：分别适配
```

不会做到所有平台 100% 相同，因为桌面端与手机端在文件系统、窗口、更新、导航、键盘和安全存储方面存在差异。

## 三、总体架构

```text id="riic67"
┌──────────────────────────────────────┐
│          React + TypeScript          │
│                                      │
│  项目管理  编辑器  大纲  人物  设置  │
└─────────────────┬────────────────────┘
                  │
                  │ Platform API
                  ▼
┌──────────────────────────────────────┐
│       TypeScript Application Core    │
│                                      │
│  小说工作流      Prompt 构建          │
│  上下文管理      Provider 适配        │
│  Token 预算      一致性规则           │
└─────────────────┬────────────────────┘
                  │
          invoke / Channel
                  ▼
┌──────────────────────────────────────┐
│            Tauri 2 / Rust            │
│                                      │
│  SQLite          文件系统             │
│  API Key         后台任务             │
│  网络请求        导入导出             │
│  权限管理        系统集成             │
└─────────────────┬────────────────────┘
                  │
        可选远程或桌面扩展
                  ▼
┌──────────────────────────────────────┐
│ Python Server / Python Sidecar       │
│ 本地模型、NLP、Embedding、高级分析    │
└──────────────────────────────────────┘
```

系统分为四个核心层：

1. React 表现层
2. TypeScript 应用与领域层
3. 平台抽象层
4. Tauri/Rust 原生能力层
