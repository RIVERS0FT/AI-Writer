## 二十二、导入和导出

首期支持：

```text id="pkpa2m"
导入：
TXT
Markdown
AI-Writer 项目文件

导出：
TXT
Markdown
JSON 项目备份
```

后续支持：

```text id="o8y7lb"
DOCX
EPUB
PDF
HTML
```

项目备份格式：

```text id="o7b5eo"
novel.aiwriter
```

本质可以是 ZIP：

```text id="9khxca"
manifest.json
project.json
chapters/
characters/
world/
assets/
```

默认不包含 API Key。

## 二十三、测试体系

### TypeScript 单元测试

使用 Vitest：

```text id="8egfb0"
Prompt 构建
Token 预算
上下文裁剪
工作流状态机
Provider 参数转换
一致性规则
```

### Rust 单元测试

使用 Cargo Test：

```text id="l5883c"
Repository
数据库迁移
密钥引用
文件导出
错误转换
Provider 流解析
```

### React 组件测试

```text id="qwqdz0"
章节编辑器
人物表单
模型配置
生成任务状态
```

### 端到端测试

使用 Playwright 或 Tauri WebDriver：

```text id="ttz3bk"
创建小说
配置模型
生成大纲
生成章节
取消任务
恢复版本
导出小说
```

模型测试不要始终调用真实 API，应提供 Mock Provider。

## 二十四、构建与发布

开发环境需要：

```text id="mmpl4c"
Node.js
pnpm
Rust
平台编译工具链
```

Android 还需要 Android Studio、Android SDK 和 NDK；iOS 构建需要 macOS 和 Xcode。([v2.tauri.app](https://v2.tauri.app/start/prerequisites/))

发布目标：

```text id="e7yyzo"
Windows：
MSI / NSIS

macOS：
DMG / App Bundle

Linux：
AppImage / DEB / RPM

Android：
AAB / APK

iOS：
App Store Archive

Web：
静态站点 / PWA
```

Tauri 官方文档提供桌面安装包、Google Play 和 App Store 的分发流程。([v2.tauri.app](https://v2.tauri.app/distribute/))

CI/CD 推荐 GitHub Actions：

```text id="w7hcj4"
pull_request：
Lint
Type Check
Unit Test
Rust Test

release tag：
Windows 构建
macOS Intel 构建
macOS ARM 构建
Linux 构建
生成发布产物
```

Android 和 iOS 建议单独建立发布工作流，因为签名和商店凭据管理更复杂。
