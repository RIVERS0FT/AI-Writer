# Token 仅统计、不限制决策

## 决策

AI-Writer 不设置应用层 Token 配额。Token 数据只用于统计、审计、性能分析、模型容量判断和可选费用估算。

## 不实现的功能

- 单任务 Token 上限
- 单章节 Token 上限
- 项目、每日或每月额度
- 因累计用量过高自动停止
- 因预计费用过高拒绝生成
- 因 Token 用量跳过规划、审查、改写或记忆提取
- 固定比例上下文预算

## 唯一容量边界

模型声明的上下文窗口是请求的技术容量边界。只有预计输入与请求输出超过该窗口时，系统才能压缩上下文或提示更换模型。

容量处理优先去重和摘要，不能删除当前要求、锁定设定、当前大纲和硬约束。

## 统计来源

Token 统计分为：

- `provider`：模型服务精确返回
- `estimated`：官方 tokenizer 或本地算法估算
- `unknown`：无法获得

未知值必须保存为 `NULL` 或 `undefined`，不能使用零代替。

## 重试规则

每次请求和每次重试单独统计。任务总用量是所有请求的汇总，包括失败请求和重试请求。

## 费用

费用只能在存在可靠价格信息时估算，并明确标记价格来源。费用统计不得影响任务执行。

## 可观测性

系统应支持按以下维度查询：

- 请求
- 写作步骤
- 写作任务
- 章节
- 项目
- 模型档案
- 时间范围

统计页面不展示额度进度条、剩余额度或自动停用状态。
