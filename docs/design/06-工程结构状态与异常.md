## 二十、状态管理

推荐职责划分：

### Zustand

保存纯前端交互状态：

```text id="na4i9j"
当前项目
当前章节
侧边栏状态
选中文本
编辑器状态
布局状态
临时生成文本
```

### TanStack Query

管理异步数据：

```text id="2h15ki"
项目列表
章节内容
人物列表
模型列表
任务状态
使用统计
```

### SQLite

保存持久化业务数据。

不要把整个小说项目长期保存在 Zustand 中，否则容易出现内存占用、状态同步和数据丢失问题。

## 二十一、异常和重试

统一错误类型：

```ts id="g5sjpw"
type AppErrorCode =
  | "NETWORK_ERROR"
  | "AUTH_ERROR"
  | "RATE_LIMITED"
  | "MODEL_NOT_FOUND"
  | "CONTEXT_TOO_LONG"
  | "GENERATION_CANCELLED"
  | "DATABASE_ERROR"
  | "FILESYSTEM_ERROR"
  | "INVALID_RESPONSE";
```

模型请求重试策略：

```text id="jrfemo"
网络临时失败：指数退避重试
429 限流：读取 Retry-After
401/403：不自动重试
上下文过长：重新裁剪上下文
用户取消：立即终止
结构化输出错误：允许修复性重试
```

每次生成都记录：

- Provider
- 模型
- 开始时间
- 结束时间
- 输入 Token
- 输出 Token
- 重试次数
- 失败原因
- 用户是否取消

日志不能记录完整 API Key，也应允许用户关闭详细 Prompt 日志。
