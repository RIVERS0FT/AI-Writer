# 第八阶段写作任务与 Token 统计界面状态

## 本阶段目标

把统一写作任务、步骤和请求级 Token 数据展示给用户，提供统计与审计能力，不增加任何额度或自动停止逻辑。

## 已实现

- “写作统计”入口
- 小说项目切换
- 项目累计 Token 汇总
- 最近写作任务列表
- 写作任务状态、类型、模型档案和更新时间
- 单任务输入、输出和总 Token
- 单任务请求数、重试数和未知请求数
- 写作步骤列表
- 步骤状态和尝试次数
- 步骤输入、输出和总 Token
- 步骤 Token 来源与延迟
- Prompt ID 和版本显示
- Provider 请求明细表
- 请求步骤、尝试序号、状态、模型、Token、来源和延迟
- `provider` 显示为“精确”
- `estimated` 显示为“估算”
- `unknown` 显示为“未知”
- 手动刷新
- 桌面与窄屏响应式布局

## 数据来源

```text
项目累计
  -> UsageRepository.summarizeProject

任务累计
  -> UsageRepository.summarizeTask

步骤列表
  -> WritingRepository.listSteps

请求明细
  -> UsageRepository.listForJob
```

界面只读取已经持久化的统计事实，不自行推测 Token，也不改变生成任务状态。

## Token 展示原则

- 未知值显示“未知”，不显示为零。
- 精确、估算和未知请求分别计数。
- 失败和重试请求仍保留在明细中。
- 项目和任务总量包含失败请求产生的已知 Token。
- 不显示额度进度条、剩余额度或超限警告。

## 当前限制

- 尚未提供章节维度和模型维度的独立筛选页面。
- 尚未提供时间范围筛选和趋势图。
- 尚未显示费用估算。
- 当前刷新为手动触发，没有任务完成后的实时推送。
- 请求错误详情目前只保存在仓储，表格尚未展开显示。
- 尚未进行桌面端人工视觉验收。

## 下一阶段

1. 增加章节、模型和时间范围筛选。
2. 增加 Token 趋势图和步骤占比。
3. 展示失败请求错误详情。
4. 任务完成后自动刷新统计。
