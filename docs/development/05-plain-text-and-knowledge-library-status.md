# 第五阶段纯文本与小说资料库实现状态

## 决策变更

项目不支持富文本。章节正文、人物资料、世界观和大纲均以纯文本为唯一编辑形式。

章节与章节版本已经完成单字段收缩，只保留 `plain_text`。旧的 JSON 正文和 Markdown 正文列已通过数据库迁移删除。

## 已实现

- 移除 Tiptap React 和 Starter Kit 依赖
- 章节编辑器替换为原生纯文本输入区
- 纯文本成为章节唯一正文
- AI 续写只请求并追加纯文本
- 生成任务恢复只追加纯文本
- 手动版本和自动保存只写入纯文本
- SQLite 迁移删除两个旧正文列
- TypeScript 领域类型删除两个旧正文属性
- Zod 保存输入删除两个旧正文属性
- Web 本地存储自动移除旧属性
- 人物领域模型和跨平台 CRUD
- 人物姓名、别名、设定、动机和当前状态
- 世界观条目领域模型和跨平台 CRUD
- 大纲节点领域模型和跨平台 CRUD
- 全书、卷、章、场景和备注节点类型
- 资料锁定标记
- 原生 SQLite 资料库仓储
- Web 本地存储资料库仓储
- 小说资料库管理界面
- 纯文本迁移单元测试

## 纯文本数据流

```text
章节打开
  -> 读取 plain_text
  -> 原生 textarea 编辑
  -> 900ms 防抖保存
  -> 更新 plain_text 与 content_hash
  -> 版本快照仅保存 plain_text
```

## 资料库数据流

```text
选择小说项目
  -> 加载人物 / 世界观 / 大纲
  -> 新建或编辑纯文本资料
  -> Zod 校验
  -> SQLite / localStorage
  -> 后续 ContextBuilder 和记忆系统读取
```

## 权威锁定

人物、世界观和大纲条目均支持锁定。锁定表示该条目是作者确认的权威资料。下一阶段的自动记忆提取只能提出修改建议，不能直接覆盖锁定内容。

## 当前限制

- 旧格式转换器只处理常见段落、换行、列表和实体。
- 资料库首版采用列表和表单，没有树形拖拽大纲。
- 大纲父子关系已经建模，界面暂未开放层级选择。
- 原生端资料删除当前为直接删除；统一回收站将在后续实现。
- 资料库尚未自动加入模型上下文。
- 章节保存尚未触发结构化记忆提取。
